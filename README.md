# АСУ «Рекрут+»

**АСУ «Рекрут+»** — це веб-система для автоматизації процесів військового рекрутингу. Проєкт призначений для ведення електронної бази кандидатів, створення анкет, обліку документів, контролю етапів відбору, перегляду аналітики, керування користувачами, ролями доступу та журналом системних дій.

Система реалізована як вебзастосунок із клієнтською частиною, REST API та базою даних PostgreSQL.

---

## Призначення проєкту

Метою проєкту є створення зручного та структурованого інструменту для автоматизації роботи рекрутингового підрозділу.

Система дозволяє:

- реєструвати кандидатів;
- створювати та редагувати анкети;
- зберігати контактні, освітні та службові дані кандидатів;
- завантажувати та перевіряти документи;
- змінювати статуси та етапи відбору;
- переглядати аналітичні показники;
- керувати користувачами системи;
- налаштовувати ролі та права доступу;
- фіксувати дії користувачів у журналі.

---

## Основні можливості

### Dashboard

Головна сторінка системи відображає ключові показники рекрутингу:

- загальна кількість кандидатів;
- нові кандидати за тиждень;
- кандидати на співбесіді;
- рекомендовані кандидати;
- графік динаміки реєстрацій;
- розподіл кандидатів за статусами.

### База кандидатів

Розділ кандидатів дозволяє переглядати, шукати, фільтрувати, редагувати та архівувати записи кандидатів.

Для кожного кандидата зберігаються:

- ID кандидата;
- ПІБ;
- дата народження;
- телефон;
- email;
- область і місто;
- освіта;
- досвід роботи;
- військовий досвід;
- бажаний підрозділ;
- поточний статус;
- поточний етап відбору.

### Нова анкета

Форма створення кандидата реалізована у вигляді покрокового сценарію:

1. Особисті дані.
2. Контакти.
3. Освіта та досвід.
4. Документи.
5. Підтвердження.

Такий підхід дозволяє зручно вводити великий обсяг інформації та зменшує кількість помилок під час заповнення.

### Документи

Розділ документів використовується для обліку файлів кандидатів.

Підтримуються такі типи документів:

- паспорт;
- ІПН;
- диплом;
- медична довідка;
- військовий квиток;
- інші документи.

Для кожного документа можна зберігати статус перевірки: очікує перевірки, підтверджено або відхилено.

### Статуси та етапи

Система підтримує контроль проходження кандидата через етапи відбору:

- первинна реєстрація;
- перевірка документів;
- співбесіда;
- професійне тестування;
- медична перевірка;
- очікування рішення;
- рекомендовано;
- зараховано;
- відхилено.

Зміна статусу або етапу фіксується в історії кандидата та журналі системних дій.

### Аналітика

Аналітичний модуль дозволяє оцінювати ефективність рекрутингу за основними показниками:

- динаміка залучення кандидатів;
- розподіл кандидатів за статусами;
- ефективність рекрутерів;
- конверсія відбору;
- кількість рекомендованих кандидатів.

### Користувачі та ролі доступу

У системі реалізовано рольову модель доступу.

Основні ролі:

- Адміністратор;
- Рекрутер;
- Керівник;
- Оператор.

Кожна роль має власний набір прав для перегляду, створення, редагування, архівації, роботи з документами, перегляду аналітики та керування системою.

### Журнал дій

Усі важливі дії користувачів фіксуються в журналі:

- створення кандидата;
- редагування анкети;
- зміна статусу;
- зміна етапу;
- завантаження документа;
- підтвердження або відхилення документа;
- створення користувача;
- зміна ролей доступу;
- експорт звітів.

## Технологічний стек

### Frontend

- HTML5;
- CSS3;
- JavaScript;
- React;
- Vite;
- Canvas-графіки;
- адаптивний інтерфейс.

### Backend

- Node.js;
- Express;
- REST API;
- middleware для службової логіки;
- завантаження файлів через Multer;
- робота з PostgreSQL через `pg`.

### База даних

- PostgreSQL;
- SQL-міграції;
- довідники статусів, ролей, прав і типів документів;
- індекси для оптимізації пошуку.

### Інфраструктура

- Docker;
- Docker Compose;
- Git;
- `.env` конфігурація.
