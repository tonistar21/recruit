# План реалізації «Рекрут+»

## Стратегія міграції

Початкова статична версія збережена у `legacy/`. Нова система будується як монорепозиторій із незалежними React frontend та FastAPI backend. Бізнес-дані більше не зберігаються у браузері.

1. Зафіксувати legacy та архітектурні рішення.
2. Створити PostgreSQL-модель, Alembic-міграцію та development seed.
3. Реалізувати cookie-based authentication, CSRF, сесії, RBAC та аудит.
4. Створити React shell, маршрути, теми й layouts для персоналу та кандидата.
5. Підключити кандидати, документи, workflow, співбесіди, сповіщення й аналітику до REST API.
6. Додати тести, Docker Compose та експлуатаційну документацію.
7. Виконати чистий запуск і зафіксувати результати у `FINAL_REPORT.md`.

## Правила приймання

Функція вважається готовою лише після реалізації backend-перевірки доступу, frontend-станів помилки/завантаження та відповідної автоматичної або ручної перевірки.
